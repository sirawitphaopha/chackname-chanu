# CLAUDE.md — คู่มือสำหรับ AI ช่วยพัฒนาเว็บนี้

ไฟล์นี้ให้ Claude (หรือ AI อื่น) อ่านก่อนแก้โค้ด เพื่อเข้าใจโครงสร้างและกฎของโปรเจกต์

## โปรเจกต์นี้คืออะไร

เว็บแอปเช็คชื่อเข้าเรียนรายคาบของ **โรงเรียนชานุมานวิทยาคม** รีโครงจากระบบเดิม
(Google Apps Script + Google Sheet, เข้ารหัสสถานะแบบ `A1B2`) มาเป็นเว็บหน้าเดียว
เก็บ localStorage และเตรียมโครงเชื่อม Supabase

## โครงสร้างไฟล์

- **`index.html`** — ทั้งเว็บอยู่ในไฟล์เดียว เรียงเป็น 3 ส่วน:
  1. `<style>` — CSS ทั้งหมด (ตัวแปรสีอยู่ที่ `:root`)
  2. HTML ของหน้า (header → controls → legend → summary → grid การ์ด → toolbar ล่าง)
  3. `<script>` — JS ทั้งหมด (ไม่มี framework, vanilla JS)
- `supabase-schema.sql` — ตาราง `students` + `attendance` (สถานะเก็บตรงๆ ไม่เข้ารหัส)
- `sw.js`, `manifest.webmanifest`, `icon.svg` — PWA

## แนวคิดหลักในโค้ด (index.html)

- **`DataService`** — ชั้นข้อมูล มี 3 เมธอด: `getStudents`, `getAttendance`, `saveAttendance`
  ตอนนี้ใช้ localStorage แต่ละเมธอดมีบล็อกคอมเมนต์ `===== SUPABASE (ยังไม่เปิดใช้) =====`
  ที่เขียนโค้ด Supabase จริงไว้พร้อมสลับ **แก้เรื่องข้อมูล = แก้ที่นี่ที่เดียว**
- **สถานะนักเรียน** มี 6 ค่า: `unmarked` (เทา, ค่าเริ่มต้น), `present`, `absent`, `late`, `leave`, `sick`
  เซ็ตผ่าน attribute `data-status` บนการ์ด `.scard` (CSS ทาสีตาม `data-status`)
- **`persist()`** — บันทึกอัตโนมัติ เรียกทุกครั้งที่เปลี่ยนสถานะ (ไม่มีปุ่มบันทึกแล้ว)
- **`buildMockStudents()`** — สร้างนักเรียนจำลอง 40 คน/ห้อง (จนกว่าจะเชื่อม Supabase)
- วิชา/รหัสวิชา เก็บ localStorage key `subj_<level>_<room>_<period>`

## กฎการทำงานที่ต้องรักษา (สำคัญมาก)

1. **ภาษา UI = ไทยล้วน และห้ามมีเครื่องหมาย `?`** ในข้อความ UI ใดๆ (ปุ่ม/หัวข้อ/label/popup)
   เช่น ใช้ "ยืนยันออกจากระบบ" ไม่ใช่ "ยืนยันออกจากระบบ?"
2. **ห้าม `alert()` / `confirm()` / `prompt()` ของเบราว์เซอร์** — ใช้ toast หรือ popup ในแอปแทน
3. **แก้สไตล์มือถือ = แก้ใน `@media (max-width: 640px)` เท่านั้น** อย่าไปแตะ CSS หลักที่กระทบจอคอม
4. **การ์ด grid ใช้ auto-fill** `repeat(auto-fill, minmax(160px, 1fr))` — ปรับจำนวนคอลัมน์เองตามจอ
   (การ์ดไม่แคบกว่า 160px ทุกขนาดจอ) ห้ามกลับไปใช้จำนวนคอลัมน์ตายตัว มันจะบีบในจอช่วงกลาง
5. **ค่าเริ่มต้นการ์ด = `unmarked` (เทา)** อย่าให้ default เป็น present

## การรันดูผล / เทส

- ต้องเปิดผ่าน **localhost** (ไม่ใช่ `file://`) ไม่งั้น PWA/Service Worker ไม่ทำงาน
  เช่น `npx serve .` หรือ `python -m http.server`
- **ระวัง Service Worker แคชหน้าเก่า** — ถ้าแก้แล้วหน้าไม่เปลี่ยน ให้ทำอย่างใดอย่างหนึ่ง:
  - เพิ่มเลขเวอร์ชันใน `sw.js` (`const CACHE = 'checkin-vN'`) แล้วรีเฟรช
  - หรือล้าง cache + unregister SW ผ่าน DevTools แล้วรีเฟรช

## แนวทางแก้โค้ด

- ไฟล์เดียว vanilla JS — แก้ตรงจุด อย่าเพิ่ม framework/build step
- ตัวแปรสีทั้งหมดอยู่ที่ `:root` — เปลี่ยนธีมแก้ที่นั่น
- ก่อนแก้เยอะ ควรเข้าใจว่าจุดที่แก้กระทบจอคอมหรือมือถือ (ดูกฎข้อ 3)

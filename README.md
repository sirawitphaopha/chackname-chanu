# ระบบเช็คชื่อเข้าเรียน — โรงเรียนชานุมานวิทยาคม

เว็บแอปเช็คชื่อเข้าเรียนรายคาบ **รีโครงใหม่ทั้งหมด** จากระบบเดิม (Google Apps Script + Google Sheet)
มาเป็นเว็บหน้าเดียวที่สะอาดตา ใช้ง่าย รองรับมือถือ และเตรียมโครงไว้เชื่อม **Supabase** เมื่อพร้อม

> สถานะปัจจุบัน: เก็บข้อมูลลงเครื่อง (localStorage) ยังไม่เชื่อมฐานข้อมูลจริง — โครงโค้ด Supabase เขียนเตรียมไว้หมดแล้ว

---

## วิธีเปิดใช้งาน

### แบบเร็ว (ดูหน้าตา)
ดับเบิลคลิกไฟล์ `index.html` เปิดในเบราว์เซอร์ได้เลย ไม่ต้องติดตั้งอะไร

### แบบเต็ม (ให้ PWA / ออฟไลน์ทำงาน)
ต้องเปิดผ่านเซิร์ฟเวอร์ (ไม่ใช่ `file://`) เช่น
```bash
# ถ้ามี Node
npx serve .
# หรือถ้ามี Python
python -m http.server 8000
```
แล้วเปิด `http://localhost:8000`

ข้อมูลเก็บลงเครื่อง (localStorage) — **ติ๊กแล้วบันทึกทันที** เปิดใหม่/รีเฟรชข้อมูลยังอยู่

## ฟีเจอร์หลัก

- **การ์ดนักเรียนสีเทา "ยังไม่เช็ค"** เป็นค่าเริ่มต้น — แยกชัดว่ายังไม่เช็ค vs เช็คแล้ว
- **แตะการ์ด → เลือกสถานะได้ตรงๆ** (มา / ขาด / สาย / ลา / ป่วย) ผ่านแผงเลือกด้านล่าง ไม่ต้องแตะวนหลายที
- **ปุ่ม "มาทั้งห้อง"** แบบสลับ (กดติ๊กทั้งห้อง / กดซ้ำล้างทั้งห้อง)
- **บันทึกอัตโนมัติ** — ติ๊กปุ๊บบันทึกลงเครื่องปั๊บ ไม่มีปุ่มบันทึกให้ลืมกด
- **ค้นหานักเรียน** ด้วยชื่อหรือเลขที่ (มีไอคอนแว่นขยาย)
- **สรุปเรียลไทม์** + แถบความคืบหน้า (มา/ขาด/สาย/ลา/ป่วย)
- **คัดลอกสรุปส่งไลน์** / **บันทึกเป็นรูปภาพ**
- **รองรับทุกขนาดจอ** — มือถือ 2 คอลัมน์, จอโน้ตบุ๊ก/จอคอมปรับจำนวนคอลัมน์อัตโนมัติ (การ์ดกว้างขั้นต่ำ 160px ไม่บีบ)
- **PWA** — ติดตั้งลงมือถือ/คอมได้เหมือนแอป (ต้อง host ผ่าน https จริงถึงติดตั้งเต็มรูป)
- **ภาษาไทยล้วน** ฟอนต์ Sarabun (มีหัว อ่านง่าย) ไม่มี popup เด้งของเบราว์เซอร์ (ใช้ toast แทน)

## สิ่งที่แก้/เพิ่มจากระบบเดิม (Google Apps Script)

### หน้าตา (UI)
- โทนสีเดียว (เขียวมิ้นต์ teal) พื้นหลังสว่าง ไม่ตีกันเหมือนเดิม (ของเดิมสี 3/10 ตีกันหมด)
- ฟอนต์ไทยสวย **Sarabun** ทั้งเว็บ
- การ์ดนักเรียนโปร่ง สูงเท่ากัน **ไม่ตัดชื่อ** อีกต่อไป

### การใช้งาน (UX)
- ของเดิม: แตะการ์ด**วน**เปลี่ยนสถานะ (ขาด→มา→ลา→ป่วย→สาย→ขาด...) กว่าจะถึงสถานะที่ต้องการต้องแตะหลายที
- ของใหม่: แตะการ์ด → **เลือกสถานะที่ต้องการตรงๆ** ทีเดียวจบ
- เพิ่มปุ่ม "มาทั้งห้อง", ค้นหา, บันทึกอัตโนมัติ, ป้ายบอกสี, แถบความคืบหน้า

### ตรรกะ (Logic)
- **เลิกใช้รหัส `A1B2`** — ระบบเดิมยัดหลายคาบไว้ในเซลล์เดียวด้วยรหัส (คาบ = A–I, สถานะ = 1–5)
  ระบบใหม่เก็บสถานะ**ตรงๆ** (`'present'`, `'absent'`, `'late'`, `'leave'`, `'sick'`) 1 แถว = 1 คน/วัน/คาบ
- แก้บั๊กสีของเดิม (สี "มา" เพี้ยน, สี "สาย" เปลี่ยนตามจำนวนคน)
- เอา **login รหัส `911` ที่ hardcode** ออก
- ชื่อตัวแปร/ฟังก์ชันอ่านรู้เรื่อง แยกชั้นข้อมูล (`DataService`) ชัดเจน

> เก็บ "ตรงๆ" ทำให้ทำรายงานง่าย เช่น "เด็กคนนี้ขาดกี่ครั้งเดือนนี้" สั่ง SQL บรรทัดเดียวได้เลย ไม่ต้องถอดรหัส `A1B2` แบบระบบเดิม

## การเชื่อม Supabase (เมื่อพร้อม)

ตอนนี้ **ยังไม่เชื่อม** — เก็บลงเครื่องอย่างเดียว โครงโค้ดเตรียมไว้หมดแล้ว

1. รันไฟล์ **`supabase-schema.sql`** ใน Supabase → SQL Editor (สร้างตาราง `students`, `attendance`)
2. ในไฟล์ `index.html` เพิ่ม SDK ของ Supabase และตั้งค่าคีย์
   ```html
   <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
   <script>
     const sb = supabase.createClient('URL ของโปรเจกต์', 'anon key');
   </script>
   ```
3. ในส่วน `DataService` (มี 3 เมธอด: `getStudents`, `getAttendance`, `saveAttendance`)
   มีคอมเมนต์ `===== SUPABASE (ยังไม่เปิดใช้) =====` เขียนโค้ดไว้ให้แล้ว แค่สลับมาใช้แทน localStorage
4. เปิด RLS (ระบบกั้นสิทธิ์ข้อมูล) ตามตัวอย่างท้ายไฟล์ SQL ก่อนใช้งานจริงเสมอ

| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | ตัวเว็บทั้งหมด — HTML + CSS + JS ในไฟล์เดียว (เปิดใช้ได้เลย) |

| `sw.js` | Service Worker — แคชไฟล์ให้เปิดออฟไลน์ได้ |
| `icon.svg` | ไอคอนแอป |
| `README.md` | คู่มือนี้ |
| `CLAUDE.md` | คู่มือสำหรับ AI (Claude) เวลามาช่วยแก้โค้ด |
